Privacy Policy

Last updated: March 9, 2026

BlueLedger is a civic transparency dashboard. This policy covers the dashboard bundle currently shipped in this repository and describes what the site loads, what data it stores, and what it does not collect.

1. What the Current Dashboard Does

The current dashboard is a read-only viewer. It does not provide account creation, user profiles, direct messaging, ad targeting, in-app purchases, or user-submitted publishing flows.

2. Personal Data We Do Not Request

The shipped dashboard does not ask you to provide your name, phone number, mailing address, government ID, contact list, camera access, microphone access, or precise device location. The current bundle also does not include a login form.

3. Data Downloaded When You Load the Dashboard

After you choose to load the dashboard, BlueLedger downloads application assets plus local data files including data/officers.parquet and data/incidents.parquet. Those files are queried locally in your browser using DuckDB WebAssembly so the dashboard can render the officer and incident views.

Standard web hosting and CDN providers may still receive ordinary request metadata such as IP address, user agent, and request timing as part of serving files. BlueLedger does not use the current dashboard bundle to build advertising profiles from that information.

4. Analytics, Crash Reporting, and Tracking

The shipped dashboard bundle in this repository does not include a mobile analytics SDK, ad SDK, or crash reporting SDK. If that changes in a future release, this policy and the corresponding store disclosures must be updated before release.

5. Third-Party Network Requests

When the dashboard loads, the following third-party requests may be made:

  • Google Fonts — The dashboard stylesheet loads the Roboto Mono and Inter typefaces from fonts.googleapis.com and fonts.gstatic.com. Google may receive your IP address and user-agent string as part of serving these font files. See Google Fonts Privacy FAQ for details.
  • DuckDB WebAssembly — The DuckDB Wasm runtime used for local queries is bundled with the application assets and does not require a separate CDN fetch in the current build.

No data from your browsing session is sent to these services beyond standard HTTP request metadata. BlueLedger does not use these connections to build user profiles or collect analytics.

6. Local Processing and Storage

Dashboard queries run locally in the browser session. Your browser may temporarily cache downloaded assets according to normal browser behavior. BlueLedger does not currently expose a personal account area or long-term user data vault in this bundle.

7. Dataset Notes

The live dashboard currently displays a mix of public-source incident imports and officer records whose source family is tagged inside the bundle where it can be determined from stable identifiers. Some officer rows remain unverified or derived. See the Data Sources page and the generated data manifest for the current bundle inventory.

8. Contact

If you email us, we will use your message and email address only to respond to that request.

9. Changes to This Policy

If the shipped bundle changes materially, this page should be updated before release so it remains consistent with Play listing disclosures and the live dashboard behavior.
